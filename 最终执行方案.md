## 四、RAG 知识库详细设计

### 4.1 三层知识库结构

#### 子库 A：法规条文库

**目的：** 精确引用法条——输出时能写「依据《广告法》第九条第(三)项」

**数据需求清单：**

| 序号 | 需要什么 | 格式 | 数量 | 来源 | 优先级 |
|------|---------|------|------|------|--------|
| A1 | 《中华人民共和国广告法》全文 | TXT 或 Markdown | 全文约 70 条 | 国家法律法规数据库 flk.npc.gov.cn 或北大法宝 | ★★★ 必须 |
| A2 | 《互联网广告管理办法》相关条文 | TXT 或 Markdown | 约 20 条 | 同上 | ★★ 重要 |
| A3 | 《化妆品广告管理办法》相关条文 | TXT 或 Markdown | 约 10 条 | 同上 | ★ 可选 |

**数据格式要求：**
```
第九条 广告不得有下列情形：
（一）使用或者变相使用中华人民共和国的国旗、国歌、国徽，军旗、军歌、军徽；
（二）使用或者变相使用国家机关、国家机关工作人员的名义或者形象；
（三）使用"国家级"、"最高级"、"最佳"等用语；
...
```
不需要特殊格式，**只要是纯文本，我能自动按条文切分**。

---

#### 子库 B：处罚案例库

**目的：** 让 Agent 学会「什么样的文案被罚过」，输出时能说「参考某市监局〔2024〕XX号处罚决定」

**数据需求清单：**

| 序号 | 需要什么 | 格式 | 数量 | 来源 | 优先级 |
|------|---------|------|------|------|--------|
| B1 | 行政处罚决定书（广告违法类） | TXT/JSON/网页摘录 | 50-100 条 | 市场监管总局 gjs.samr.gov.cn → 行政处罚公示 | ★★★ 必须 |
| B2 | 典型广告违法案例汇编 | PDF/TXT | 20-30 条 | 各省市市监局官网的「典型案例」栏目 | ★★ 重要 |
| B3 | 电商广告违法案例 | 网页摘录 | 10-20 条 | 1688/淘宝规则中心、电商法务公众号 | ★★ 重要 |

**每条案例需要包含的字段（至少有前 4 个）：**

```json
{
  "title": "XX公司发布虚假广告案",
  "violation_text": "在商品详情页使用'全网销量第一'等极限用语",
  "law_refs": "《广告法》第九条第(三)项、第五十七条",
  "penalty": "罚款人民币20万元",
  "industry": "电商",
  "violation_type": "极限用语",
  "source_url": "https://gjs.samr.gov.cn/...",
  "penalty_amount": 200000
}
```

**采集建议：**
- 市场监管总局官网的行政处罚公示栏可以按「广告违法」筛选
- 搜索引擎搜「广告法行政处罚案例 2024」「极限用语罚款案例」也能找到很多
- 公众号「广告法实务」「市监沙龙」等经常整理案例
- **你只需要把网页内容复制粘贴给我，格式化和结构化我来做**

#### 子库 C：行业规则库

**目的：** 给用户可操作的「合规替代建议」，不只是说「你违规了」

**数据需求清单：**

| 序号 | 需要什么 | 格式 | 数量 | 来源 | 优先级 |
|------|---------|------|------|------|--------|
| C1 | 广告禁用词清单 | TXT/CSV/JSON | 200+ 词 | GitHub 搜「广告禁用词」「极限用语清单」 | ★★★ 必须 |
| C2 | 禁用词→合规替代词映射 | CSV/JSON | 100+ 对 | 法务公众号整理、市监局宣传材料 | ★★★ 必须 |
| C3 | 各行业广告特殊禁区 | TXT/JSON | 5-10 个行业 | 行业协会规范、平台规则（淘宝规则中心等） | ★★ 重要 |

**C1 禁用词清单示例：**

```
虚假宣传：100%有效、根治、永不反弹、一次见效、包治百病、假一赔万...
对比贬低：秒杀同行、吊打XX品牌、碾压竞品...
承诺收益：稳赚不赔、保本保收益、买了就能赚...
```

**C2 禁用→替代映射示例：**

**C3 行业禁区示例：**

### 4.2 vmem 存储策略

用你的 cc-hms 的 `MemoryVectorStore`，按 `source` 字段区分三个子库：

```python
# 法规条文 → source="law"
store.store(
    key="ad_law_art9_3",
    value="第九条第(三)项 广告不得使用国家级、最高级、最佳等用语",
    source="law",
    tags="极限用语,广告法,第九条"
)

# 处罚案例 → source="case"
store.store(
    key="case_2024_001",
    value="XX公司因使用'全网销量第一'被罚款20万元",
    source="case",
    tags="极限用语,电商,罚款20万"
)

# 行业规则 → source="rule"
store.store(
    key="rule_food_no_medical",
    value="食品广告不得宣称疾病预防治疗功能，不得使用医疗术语",
    source="rule",
    tags="食品行业,禁区规则"
)
```

检索时用 `search_fusion` 并通过 `source_filter` 区分子库。

## 六、可靠性工程

### 防线 1：溯源

每条违规输出附带：
```json
{
  "position": "全网最低价",
  "violation_type": "极限用语",
  "law_reference": "广告法第九条第(三)项",      // ← 溯源到具体法条
  "case_reference": "XX公司罚款案(2024)",       // ← 溯源到类似案例
  "risk_level": "高"
}
```

### 防线 2：结构化输出

```python
class ViolationItem(BaseModel):
    position: str           # 违规位置（原文引用）
    violation_type: str     # 违规类型
    law_reference: str      # 法条引用
    case_reference: str     # 类似案例引用（可选）
    risk_level: str         # 高/中/低
    confidence: float       # Agent 的置信度 0-1
    suggestion: str         # 替代建议

class ComplianceReport(BaseModel):
    violations: list[ViolationItem]
    overall_risk: str
    summary: str
    total_violations: int
    sources: list[str]      # 所有引用的知识来源
```

### 防线 3：人在回路

```
Agent 输出审查报告草稿
    ↓
界面展示：每个违规点一个卡片
    ↓
用户操作：[✅ 确认] [✏️ 修改] [❌ 驳回]
    ↓
确认 → 最终报告
修改 → 用户编辑后存入最终报告
驳回 → 记录到反馈日志（vmem store，source="feedback"）
    ↓
反馈日志积累后 → 置信度更新 → Agent 越用越准
```

**驳回的反馈存储：**
```python
store.store(
    key=f"feedback_{timestamp}",
    value="文案'买到就是赚到'被运营驳回，判断为口语化表达不构成承诺收益",
    source="feedback",
    tags="驳回,承诺收益,误判"
)
```

下次 Agent 遇到类似语境时，`search_fusion` 会检索到这条反馈，结合置信度调整判断。
