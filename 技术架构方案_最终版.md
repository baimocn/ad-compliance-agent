#### 修复 2：FTS 评分引入分词（验证工程师 RISK 项）

`storage.py` 中替换逐字符匹配：

```python
import jieba

def _compute_fts_score(self, query: str, text: str) -> float:
    query_words = set(jieba.cut(query)) - {" ", "\n"}
    text_words = set(jieba.cut(text)) - {" ", "\n"}
    if not query_words:
        return 0.0
    return len(query_words & text_words) / len(query_words)
```

#### 修复 3：反馈检索增加 topic_filter（验证工程师 RISK 项）

`retrieval.py` 中 `_apply_feedback_boost` 改为逐条检索：

```python
async def _apply_feedback_boost(self, results: list[dict], query: str) -> list[dict]:
    for result in results:
        key = result["key"]
        feedback_results = self.store.search_fusion(
            query=f"feedback:{key}",
            top_k=10,
            w_vec=0.2, w_fts=0.6, w_time=0.1, w_conf=0.1,
            source_filter=["feedback"],
            topic_filter=[f"target:{key}"],   # 新增：精确过滤
        )
        # 应用 feedback boost/penalty 到 result["score"]
        for fb in feedback_results:
            if "accepted" in fb.get("tags", []):
                result["score"] *= 1.5
            elif "rejected" in fb.get("tags", []):
                result["score"] *= 0.3
            elif "modified" in fb.get("tags", []):
                result["score"] *= 0.5
    return results
```

#### 修复 4：VmemStore 增加公共接口（验证工程师 RISK 项）

```python
def update_confidence(self, key: str, delta: float) -> None:
    """公共接口，替代直接操作 _storage"""
    if key in self._storage:
        current = self._storage[key].get("confidence", 0.5)
        new_val = max(0.1, min(1.0, current + delta))
        self._storage[key]["confidence"] = new_val
```

```python
async def batch_judge(
    self,
    text: str,
    hits: list[BannedWordHit],
    product_category: str | None = None,
    max_concurrent: int = 3,    # 新增：匹配 DeepSeek API QPS 限制
) -> list[ContextJudgment]:
    semaphore = asyncio.Semaphore(max_concurrent)

    async def limited_judge(hit: BannedWordHit) -> ContextJudgment:
        async with semaphore:
            return await self.judge(text, hit, product_category)

    tasks = [limited_judge(hit) for hit in hits]
    results = await asyncio.gather(*tasks, return_exceptions=True)

    judgments: list[ContextJudgment] = []
    for i, result in enumerate(results):
        if isinstance(result, Exception):
            judgments.append(ContextJudgment(
                is_violation=True,
                reasoning=f"语境判断服务异常，默认标记为违规。错误：{str(result)[:80]}",
                confidence=0.0,
            ))
        else:
            judgments.append(result)
    return judgments
```

#### 修复 9：生成 highlightedText 和 violation positions

```python
# backend/lib/html_highlight.py
import re

def generate_highlighted_html(original_text: str, violations: list) -> str:
    """
    根据 violations 的 startIndex/endIndex 生成带 <mark> 标签的 HTML。
    先按位置排序，再逐段拼接，对用户输入做 HTML 转义。
    """
    import html as html_lib

    sorted_v = sorted(violations, key=lambda v: v.start_index)
    parts: list[str] = []
    cursor = 0

    for v in sorted_v:
        # 非违规部分
        if cursor < v.start_index:
            parts.append(html_lib.escape(original_text[cursor:v.start_index]))
        # 违规部分：转义后包裹 <mark>
        escaped = html_lib.escape(original_text[v.start_index:v.end_index])
        parts.append(
            f'<mark data-vid="{v.id}" '
            f'class="bg-red-200 dark:bg-red-900 rounded px-0.5 cursor-pointer '
            f'underline decoration-wavy decoration-red-500">{escaped}</mark>'
        )
        cursor = v.end_index

    # 尾部
    if cursor < len(original_text):
        parts.append(html_lib.escape(original_text[cursor:]))

    return "".join(parts)
```

此函数对用户输入做 `html.escape()`，防止 XSS。`HighlightedText` 前端组件用 `dangerouslySetInnerHTML` 渲染时无需额外 DOMPurify。

### 2.6 降级策略汇总

| 失败场景 | 降级策略 |
|---------|---------|
| 禁用词典文件不存在或格式损坏 | `RuntimeError`，启动失败，快速报错 |
| DeepSeek API 连通性测试失败 | 启动时 lifespan 中检测，log warning 但不阻塞启动 |
| LLM 语境判断超时/异常 | `asyncio.Semaphore(3)` 限流 + `tenacity` 3 次重试 + 默认标记违规 |
| LLM 返回空/拒绝/格式错误 | Pydantic ValidationError 重试一次，再失败默认违规 |
| vmem 检索无结果 | 返回空 citations |
| vmem 服务不可用 | catch 异常，返回空 citations + log |
| 替代文案生成失败 | 返回空 rewrite_options |
| 替代文案自检不通过 | 兜底返回所有方案 |
| review_id 碰撞 | 使用完整 UUID hex[:8]（32 bit） |
| 输入文案为空/超长 | agent.review() 入口显式校验 |

```typescript
// src/hooks/useReviewSubmit.ts
export function useReviewSubmit(machine: ReturnType<typeof useReviewMachine>) {
  const { state, actions } = machine;
  const abortRef = useRef<AbortController | null>(null);

  const handleSubmit = useCallback(async () => {
    if (!state.text.trim()) return;
    actions.submit();

    const controller = new AbortController();
    abortRef.current = controller;

    try {
      const result = await submitReview({
        text: state.text,
        industry: state.industry as Industry,
        platform: state.platform || undefined,
      }, controller.signal);   // 传递 signal
      actions.submitSuccess(result);
    } catch (err) {
      if (controller.signal.aborted) return;  // 用户取消，不报错
      actions.submitError(
        err instanceof Error ? err.message : "审查请求失败，请重试"
      );
    }
  }, [state.text, state.industry, state.platform, actions]);

  const handleCancel = useCallback(() => {
    abortRef.current?.abort();
    actions.cancel();
  }, [actions]);

  return { handleSubmit, handleCancel, isSubmitting: state.phase === "submitting" };
}
```

### 4.1 文件结构

### 4.2 评测维度权重

**Agent 评测 (30 题)：**

| 维度 | 权重 | 说明 |
|------|------|------|
| issue_recall | 35% | 是否识别出所有违规项 |
| risk_accuracy | 25% | 风险等级判定是否准确 |
| law_citation | 25% | 法条引用是否正确 |
| suggestion_quality | 15% | 替代建议是否合理 |

**RAG 评测 (15 题)：**

| 维度 | 权重 | 说明 |
|------|------|------|
| key_recall | 45% | 是否检索到正确文档 |
| source_recall | 30% | 是否命中正确的子库 |
| precision | 25% | 返回结果中相关文档占比 |

| 指标 | Mock Baseline | 目标值 |

| Agent 平均分 | 0.301 | >= 0.7 |
